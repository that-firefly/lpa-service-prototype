Audit findings and assumptions

Navigation and header

The original design included a secondary navigation bar beneath the header, adding visual complexity. This was replaced with a consolidated menu button in the header inspired by the GOV.UK One Login pattern, where a mega menu contains all navigation items, reducing cognitive load and keeping the interface focused on the task.

An Alpha phase banner was added with a feedback link to clearly communicate to users that this is a new service in testing, following GOV.UK phase banner guidance.

Information architecture

The original screen presented information in a single long page adding a lot of complexity and cognitive load. There were two conflicting graphical elements with higher visual hierarchy, "Who is the donor for this LPA?" and the "Add donor details" call to action button, which could cause confusion to the user.

The introductory content was restructured to clearly communicate the purpose of the service upfront.

A details component was introduced to give users the ability to get more information about LPAs. An inset text component was added to highlight "This LPA covers property and financial affairs", making it visually distinct and easier to scan. A "Use this service to" bullet list was added to set clear user expectations before starting the journey.

One thing per page

The original single page combined service information and the donor details form, creating an overloaded entry point. This was separated into two distinct pages, a start page and an eligibility page, following the GOV.UK onboarding pattern.

The modal popup was replaced with a series of individual pages:

  • What are your details?
  • What is your date of birth?
  • What is your email address?
  • Find an address
  • Select an address
  • Confirm address
  • Can the donor sign or make a mark?

Modals create accessibility issues including focus management problems and are not recommended in the GOV.UK Design System. Breaking the form into individual pages reduces cognitive load, makes error recovery easier, and allows users to save progress at each step.

Three-step address pattern

A three-step address pattern was implemented following the DWP Design System find an address pattern: postcode entry, address selection, and address confirmation. A manual entry option was also provided. This reduces input errors and is a familiar pattern for UK users, meeting accessibility requirements better than a single address text field.

Check answers and confirmation pages

A check answers page was added before submission showing all entered data with "change" links back to individual pages, giving users confidence before submitting and reducing errors.

A confirmation page was added showing a reference number and next steps explaining the Office of the Public Guardian (OPG) registration process. It also allows the user to access previous and current LPAs and sets clear expectations about what happens after submission.
